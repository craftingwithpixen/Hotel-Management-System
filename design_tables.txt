=== TABLE 36 ===
Step | Actor | What happens
1 | Customer | POST /auth/register with { name, email, password }
2 | Server | Check email not already registered. Generate 6-digit OTP. Hash OTP (bcrypt). Store in user.otp + user.otpExpiry (now + 10 min). Set isVerified=false
3 | Resend | emailService.sendOTP(email, otp) fires React Email template with OTP
4 | Customer | POST /auth/verify-otp with { email, otp }
5 | Server | Compare OTP hash. Check otpExpiry not passed. Set isVerified=true, clear otp fields. Issue access token (15 min) + refresh token (7 days). Store hashed refresh in DB. Set refresh as httpOnly cookie
6 | Client | Store access token in Zustand authStore (memory only). React Router redirects to customer dashboard

=== TABLE 37 ===
Step | Actor | What happens
1 | Customer | GET /bookings/available-rooms?checkIn=&checkOut=&type= — server queries rooms where no confirmed booking overlaps the date range
2 | Customer | POST /bookings/room with { roomId, checkIn, checkOut, guestCount }
3 | Server | Re-validate room availability (race condition guard). Create Booking with status=pending. Room status stays available until payment
4 | Customer | POST /payments/create-order with { billingId }
5 | Server | Create Razorpay order (amount in paise). Return { razorpay_order_id, amount, currency, key }
6 | Client | Open Razorpay checkout modal with order details. Customer pays
7 | Customer | POST /payments/verify with { razorpay_payment_id, razorpay_order_id, razorpay_signature }
8 | Server | HMAC-SHA256 verify. Mark Payment captured. Booking status=confirmed. Room status=booked. Add loyalty points to customer
9 | Resend | emailService.sendBookingConfirmation with booking details and PDF invoice attached
10 | Socket | Emit booking:new to receptionist room

=== TABLE 38 ===
Step | Actor | What happens
1 | Waiter | Selects table on their dashboard. Adds items from menu. POST /orders with { tableId, items: [{ menuItemId, quantity, notes }] }
2 | Server | Create Order document. Set table.status=occupied. Snapshot item prices. overallStatus=pending
3 | Socket | socketService.emitNewOrder(io, order) → io.to("kitchen").emit("new:order", {...})
4 | Chef | Kitchen board auto-updates — new KOT card appears in Pending column. No page refresh needed
5 | Waiter | POST /orders/:id/kot — marks kotPrinted=true, locks items from modification
6 | Chef | Drags order to Preparing. PUT /orders/:id/status { status: "preparing" }
7 | Socket | io.to("table:{tableId}").emit("order:update") + io.to("waiter:{waiterId}").emit("order:update")
8 | Customer | If they scanned QR and are on /scan/table/:id, their status chip updates live: Pending → Preparing → Ready
9 | Chef | Marks Ready. Waiter notified. Sets Served after delivery
10 | Receptionist | POST /billing/generate — computes subtotal + GST + any discount. POST /payments/cash or online
